# Paragraph — Week 21 Threshold Comparison Interpretation

Table X shows that the final locked serving threshold (`0.1279`) is the most suitable operating point for the frozen deployed system. Compared with the historical Week 12 reference threshold (`0.0884`), it preserves the same fraud recall on the locked test set while reducing false positives, improving precision, and slightly lowering expected cost per transaction. By contrast, the post-hoc recall-first operating point demonstrates that recall above 0.90 is technically achievable only at the expense of a severe collapse in precision and an operationally unrealistic alert burden. This confirms that the final threshold decision should be interpreted as a business-aware serving choice rather than as an attempt to maximize recall at any cost.
