# Σκοπός
Διερεύνηση κατανομών `Time` & `Amount`, συνολικά και ανά κλάση (`Class=0/1`).

- Skewness Amount: 16.98 (πολύ έντονα δεξιά ασύμμετρη)
- Skewness Time: -0.036 (σχετικά συμμετρική)
- 99ο εκατοστημόριο Amount: ~1018 €

# Ευρήματα
1. **Κατανομή Amount**  
   - Εξαιρετικά ασύμμετρη προς τα δεξιά με πολλές μικρές συναλλαγές.  
   - Η log1p μετατροπή εξομαλύνει την κατανομή και κάνει το χαρακτηριστικό πιο κατάλληλο για ML αλγόριθμους.

2. **Κατανομή Time / Hour-of-Day**  
   - Συνολικά, οι συναλλαγές είναι πιο έντονες μεταξύ 9:00–21:00.  
   - Οι απάτες εμφανίζουν σχετικά υψηλότερα ποσοστά σε ώρες χαμηλής κίνησης (π.χ. 2πμ–4πμ).

3. **Διαφορές ανά κλάση**  
   - Fraud συναλλαγές εμφανίζονται πιο συγκεντρωμένες σε συγκεκριμένες ώρες.  
   - Τα ποσά απάτης συχνά είναι είτε πολύ μικρά είτε σε συγκεκριμένα spikes.

4. **Επόμενα βήματα**  
   - Επιλογή features: χρήση log1p(Amount), hour-of-day.  
   - Εστίαση σε metrics ευαίσθητα σε ανισορροπία (PR-AUC, Recall).  
   - Προετοιμασία για resampling τεχνικές (SMOTE, undersampling).

# Σχόλια & Επόμενα Βήματα
- Εξέταση resampling (SMOTE/undersampling) στην Εβδομάδα 7.
- Χρήση `hour` ως χαρακτηριστικό (feature engineering).
- Κατωφλίωση/threshold-tuning με γνώμονα Recall ≥ 0.90.
