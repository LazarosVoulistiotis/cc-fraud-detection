# Reports structure

The `reports/` folder is organized by month to keep the thesis workflow audit-friendly:

- `reports/month1/` → Problem framing, dataset overview, EDA foundations
- `reports/month2/` → Data quality, scaling, class imbalance, feature analysis
- `reports/month3/` → Modeling, threshold policy, business-oriented selection

Figures used in the written report live under `reports/figures/`.
Short report-ready snippets (tables/paragraphs) live under `reports/report_snippets/`.
